# 启动步骤

## 完整启动流程

### 1. 环境检查

当前项目后端已迁移为 Node.js + TypeScript，不再需要 Python 虚拟环境。

```bash
node --version
npm --version
```

建议使用 Node.js 18 或更高版本。

### 2. 配置 API 密钥

复制配置示例：

```bash
cd backend
cp config.example.json config.json
```

编辑 `backend/config.json`：

```json
{
  "apiKey": "sk-xxxxxxxxxxxxxxxx",
  "baseUrl": "https://api.deepseek.com",
  "model": "deepseek-chat",
  "maxTurns": 6,
  "mcpServers": [
    {
      "id": "filesystem",
      "command": "node",
      "args": ["./node_modules/@modelcontextprotocol/server-filesystem/dist/index.js", ".."],
      "cwd": "backend",
      "description": "默认文件系统 MCP 服务"
    }
  ]
}
```

也可以使用环境变量覆盖：

```bash
export DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxx"
export DEEPSEEK_BASE_URL="https://api.deepseek.com"
export DEEPSEEK_MODEL="deepseek-chat"
export MAX_AGENT_TURNS="6"
```

### 3. 安装依赖

```bash
npm --prefix backend install
npm --prefix frontend install
npm --prefix electron install
```

如果 npm 下载速度较慢，可以临时使用国内镜像：

```bash
npm --prefix backend install --registry=https://registry.npmmirror.com
npm --prefix frontend install --registry=https://registry.npmmirror.com
npm --prefix electron install --registry=https://registry.npmmirror.com
```

### 4. 启动应用

#### 方式一：一键启动（推荐）

macOS/Linux：

```bash
chmod +x start.sh
./start.sh
```

默认只启动后端和前端。需要启动 Electron 时：

```bash
START_ELECTRON=1 ./start.sh
```

Windows：

```bat
start.bat
```

#### 方式二：分别启动

终端 1 - 后端：

```bash
npm --prefix backend run dev
```

后端默认地址：`http://localhost:5001`。

终端 2 - 前端：

```bash
npm --prefix frontend run dev
```

前端默认地址：`http://localhost:3000`。

终端 3 - Electron（可选）：

开发模式：

```bash
cd electron
npm run dev
```

生产模式（需先构建前端和后端）：

```bash
cd electron
npm start
```

### 5. 验证安装

1. 打开 `http://localhost:3000` 或 Electron 窗口。
2. 创建新会话。
3. 上传一个测试文件，输入任务描述。
4. 后端健康检查地址应返回成功：`http://localhost:5001/api/health`。

## 生产构建

### 构建后端

```bash
npm --prefix backend run build
```

构建后可独立运行：

```bash
npm --prefix backend start
```

### 构建前端

```bash
npm --prefix frontend run build
```

### 打包 Electron

完整打包流程（生成可分发的安装包）：

```bash
# 1. 构建前端
npm --prefix frontend run build

# 2. 构建后端
npm --prefix backend run build

# 3. 打包 Electron（会包含前端 dist 和后端 dist）
npm --prefix electron run build
```

生成的安装包位于 `electron/dist/` 目录。

### 测试生产模式（不打包）

如果只想测试生产模式而不生成安装包：

```bash
# 1. 构建前端和后端
npm --prefix frontend run build
npm --prefix backend run build

# 2. 以生产模式启动 Electron
cd electron
npm start
```

Electron 会：
- 启动后端的生产进程（`node dist/index.js`）
- 通过健康检查等待后端就绪
- 加载前端的 `dist/index.html`

## 常见启动问题

### 后端提示缺少 DeepSeek API Key

确认 `backend/config.json` 存在并包含 `apiKey`，或确认环境变量 `DEEPSEEK_API_KEY` 已设置。

### 前端无法请求后端

确认后端运行在 `5001` 端口。开发模式下，`frontend/vite.config.js` 会把 `/api` 代理到 `http://localhost:5001`。

### 端口被占用

```bash
lsof -i :5001
lsof -i :3000
kill -9 <PID>
```

也可以通过环境变量修改后端端口：

```bash
PORT=5002 npm --prefix backend run dev
```

前端端口可通过启动参数修改：

```bash
npm --prefix frontend run dev -- --port 3001
```

### Electron 启动后看不到页面

**开发模式**（`npm run dev`）：
- 需要确保前端 Vite 开发服务器已在 `http://localhost:3000` 运行
- 后端会自动以 `npm run dev` 启动（tsx watch）

**生产模式**（`npm start`）：
- 需要先构建前端和后端：
  ```bash
  npm --prefix frontend run build
  npm --prefix backend run build
  ```
- Electron 会启动后端生产进程并等待健康检查通过后再显示窗口

### npm 依赖或缓存异常

```bash
rm -rf backend/node_modules frontend/node_modules electron/node_modules
npm --prefix backend install
npm --prefix frontend install
npm --prefix electron install
```

Windows 可手动删除对应 `node_modules` 目录后重新安装。

## 停止应用

- 使用 `start.sh` 启动时，按 `Ctrl+C` 会清理已启动的后端和前端进程。
- 分别启动时，在各终端按 `Ctrl+C`。
- Electron 可直接关闭窗口。

## 注意事项

- `backend/config.json` 包含密钥，不要提交到仓库。
- `backend/data/` 是运行时会话数据，商用部署前应设计备份、清理和数据保留策略。
- `node_modules/`、`backend/dist/`、`frontend/dist/` 都可重新生成，不应作为源码提交。
