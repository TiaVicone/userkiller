# 启动步骤

## 完整启动流程

### 第一步：环境检查

确保已安装以下软件：

```bash
# 检查Python版本（需要3.8+）
python3 --version

# 检查Node.js版本（需要16+）
node --version

# 检查npm版本
npm --version
```

### 第二步：配置API密钥

1. 打开 `backend/config.json` 文件
2. 填入您的API密钥：

```json
{
  "deepseek_api_key": "sk-xxxxxxxxxxxxxxxx",
  "deepseek_base_url": "https://api.deepseek.com",
  "qianwen_api_key": "sk-xxxxxxxxxxxxxxxx",
  "qianwen_base_url": "https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation",
  "max_iterations": 5,
  "workspace_root": "./workspaces"
}
```

**获取API密钥：**
- DeepSeek API: https://platform.deepseek.com/
- 千问API: https://dashscope.aliyun.com/

### 第三步：创建Python虚拟环境（推荐）

使用虚拟环境可以隔离项目依赖，避免与系统Python包冲突。

#### macOS/Linux

```bash
cd backend

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 激活后，命令行前会显示 (venv) 标识
```

#### Windows

```bash
cd backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate

# 激活后，命令行前会显示 (venv) 标识
```

**注意**：每次打开新终端启动后端时，都需要先激活虚拟环境。

### 第四步：安装依赖

#### 1. 安装后端依赖

确保已激活虚拟环境（命令行前有 `(venv)` 标识），然后安装依赖：

```bash
# 确保在 backend 目录下，且虚拟环境已激活
pip install -r requirements.txt
```

如果使用的是 `pip3`：
```bash
pip3 install -r requirements.txt
```

**验证安装**：
```bash
pip list  # 查看已安装的包
```

#### 2. 安装前端依赖

```bash
cd ../frontend
npm install
```

如果npm速度慢，可以使用国内镜像：
```bash
npm install --registry=https://registry.npmmirror.com
```

#### 3. 安装Electron依赖

```bash
cd ../electron
npm install
```

### 第五步：启动应用

#### 方式一：开发模式（推荐）

打开**3个终端窗口**，分别执行：

**终端1 - 后端服务：**
```bash
cd backend

# 激活虚拟环境（重要！）
source venv/bin/activate  # macOS/Linux
# 或
venv\Scripts\activate     # Windows

# 启动后端
python app.py
```

看到以下输出表示后端启动成功：
```
🚀 AI自动化办公软件后端启动中...
📁 工作区目录: ./workspaces
🔄 最大迭代次数: 5
==================================================
 * Running on http://0.0.0.0:5001
```

**终端2 - 前端开发服务器：**
```bash
cd frontend
npm run dev
```

看到以下输出表示前端启动成功：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

**终端3 - Electron应用：**
```bash
cd electron
NODE_ENV=development npm start
```

Electron窗口将自动打开。

#### 方式二：仅使用浏览器（无需Electron）

**终端1 - 后端：**
```bash
cd backend
source venv/bin/activate  # 激活虚拟环境
python app.py
```

**终端2 - 前端：**
```bash
cd frontend
npm run dev
```

然后在浏览器中打开：http://localhost:3000

#### 方式三：生产模式

**1. 构建前端：**
```bash
cd frontend
npm run build
```

**2. 启动后端：**
```bash
cd backend
source venv/bin/activate  # 激活虚拟环境
python app.py
```

**3. 启动Electron：**
```bash
cd electron
npm start
```

### 第六步：验证安装

1. 打开应用后，点击"新会话"按钮
2. 输入测试需求："你好，请介绍一下你的功能"
3. 如果收到AI回复，说明配置成功

## 常见启动问题

### 问题1：后端启动失败

**错误：`ModuleNotFoundError: No module named 'flask'`**

解决方案：
```bash
cd backend
source venv/bin/activate  # 确保激活虚拟环境
pip install -r requirements.txt
```

### 问题2：前端启动失败

**错误：`command not found: vite`**

解决方案：
```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
```

### 问题3：端口被占用

**错误：`Address already in use`**

解决方案：
```bash
# 查找占用端口的进程
lsof -i :5001  # 后端端口
lsof -i :3000  # 前端端口

# 杀死进程
kill -9 <PID>
```

或者修改端口：
- 后端：修改 `backend/app.py` 最后一行的端口号（当前为5001）
- 前端：修改 `frontend/vite.config.js` 中的 `server.port` 和 `proxy.target`

### 问题4：API调用失败

**错误：`API key is invalid`**

解决方案：
1. 检查 `backend/config.json` 中的API密钥是否正确
2. 确认API密钥有足够的额度
3. 检查网络连接

### 问题5：Python版本不兼容

**错误：`SyntaxError: invalid syntax` 或依赖包安装失败**

解决方案：

**Python版本过低（< 3.8）：**
```bash
# 安装Python 3.8-3.12
# macOS
brew install python@3.11

# Ubuntu/Debian
sudo apt install python3.11

# 重新创建虚拟环境
cd backend
rm -rf venv
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**Python版本过高（3.13+）：**

Python 3.13 太新，某些依赖包可能不兼容。推荐使用 Python 3.8-3.12：

```bash
# macOS - 安装Python 3.11
brew install python@3.11

# 重新创建虚拟环境
cd backend
rm -rf venv
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**推荐的Python版本**：Python 3.9、3.10 或 3.11

### 问题6：忘记激活虚拟环境

**错误：导入模块失败或使用了系统Python**

解决方案：
```bash
# 检查是否在虚拟环境中
which python  # macOS/Linux
where python  # Windows

# 应该显示类似：/path/to/backend/venv/bin/python

# 如果不在虚拟环境中，激活它
cd backend
source venv/bin/activate  # macOS/Linux
venv\Scripts\activate     # Windows
```

### 问题7：虚拟环境损坏

**错误：虚拟环境无法正常工作**

解决方案：
```bash
# 删除旧的虚拟环境
cd backend
rm -rf venv  # macOS/Linux
rmdir /s venv  # Windows

# 重新创建
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## 快速启动脚本

为了方便启动，可以创建启动脚本：

### macOS/Linux

创建 `start.sh`：
```bash
#!/bin/bash

echo "🚀 启动AI自动化办公软件..."

# 启动后端（使用虚拟环境）
cd backend
source venv/bin/activate
python app.py &
BACKEND_PID=$!
echo "✅ 后端已启动 (PID: $BACKEND_PID)"

# 等待后端启动
sleep 3

# 启动前端
cd ../frontend
npm run dev &
FRONTEND_PID=$!
echo "✅ 前端已启动 (PID: $FRONTEND_PID)"

# 等待前端启动
sleep 5

# 启动Electron
echo "🖥️  启动Electron应用..."
cd ../electron
NODE_ENV=development npm start

# 清理进程
echo "🛑 正在停止服务..."
kill $BACKEND_PID $FRONTEND_PID 2>/dev/null
echo "✅ 已停止所有服务"
```

使用：
```bash
chmod +x start.sh
./start.sh
```

### Windows

创建 `start.bat`：
```batch
@echo off
echo 启动AI自动化办公软件...

start cmd /k "cd backend && venv\Scripts\activate && python app.py"
timeout /t 3

start cmd /k "cd frontend && npm run dev"
timeout /t 5

cd electron
set NODE_ENV=development
npm start
```

使用：双击 `start.bat` 文件

## 退出虚拟环境

当您完成工作后，可以退出虚拟环境：

```bash
deactivate
```

命令行前的 `(venv)` 标识将消失。

## 停止应用

按 `Ctrl + C` 停止各个终端中的进程，或直接关闭Electron窗口。
